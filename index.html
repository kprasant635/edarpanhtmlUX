Login / New Registration

Citizen opens e-Darpan, enters mobile number, verifies OTP and creates or accesses the account. The portal supports secure OTP login and bilingual citizen interface. Citizen can access the portal using Sewa-Setu portal also.

AI Chatbot Assistance

AI chatbot will be available for citizen to understand required documents, application process, case status, query response and final order download steps.

Submit New Application

Citizen fills applicant details, address, purpose/subject, district, circle office and uploads required documents. AI based auto completion of the remarks will be available for citizen. WhatsApp/SMS notification will be sent to the citizen along with the acknowledgement for this action.

Track Case Status

Citizen dashboard shows case number, submitted date, current stage, action required status, in-progress cases and completed cases. WhatsApp/SMS updates are sent whenever the stage changes.

AI Summary & Key Points

Every citizen case view will show AI generated case summary, current action required, important dates, pending documents and key points in simple language.

Respond to Query

If ASST/LRS/LRA/CO asks for clarification, citizen uploads missing/clear documents and submits remarks. The query response reaches the concerned officer dashboard, and WhatsApp/SMS notification is sent for every action required case.

Receive Final Order

After CO decision, citizen receives WhatsApp/SMS acknowledgement and can download final order PDF and application receipt from the portal.

AI Chatbot + Case Summary

ASST gets AI chatbot support in dashboard and case view. AI summary and key points will highlight missing documents, citizen remarks, purpose/category, SLA status and recommended scrutiny focus. WhatsApp/SMS notification is triggered for every forward, reject, query or return action.

ASST Dashboard

ASST dashboard shows new applications, pending applications, returned/query cases and internal cases which are registered by the ASST. Citizen query replies raised by any officer can also be viewed against the case.

Preliminary Scrutiny for Citizen Cases

ASST verifies application details. WhatsApp/SMS notification is sent for each action. Following actions can be performed:

  • Forward
  • Reject
  • Send query / return to citizen if incomplete

Query to Citizen / Query Inbox

For any step, ASST can ask the citizen for more details or documents. Citizen response will come back to the ASST case view, with WhatsApp/SMS notification for both query and response.

Register e-Darpan Cases

If citizen visits office or for internal cases, ASST can register e-Darpan cases by himself from the officer portal. The applicant receives acknowledgement through WhatsApp/SMS after registration.

Purpose-Based Routing

Based on the selected case purpose/category, ASST registered cases may follow ASST → CO or ASST → LRS → LRA → CO based on the category-process flow configuration. WhatsApp/SMS notification is sent whenever the case is routed.

AI Chatbot + Case Summary

LRS gets AI chatbot support in dashboard and case view. AI summary and key points will show scrutiny observations, document readiness, priority, workload and recommended next action. WhatsApp/SMS notification is triggered for assignment, query, return and forwarding actions.

LRS Dashboard

LRS dashboard shows cases received from ASST, cases pending LRA assignment, cases already assigned to LRA, LRA reports received, citizen query responses, cases reverted from LRA, cases returned to ASST and cases forwarded to CO. WhatsApp/SMS notification is sent for every action.

Review ASST Scrutiny

LRS receives scrutiny-passed cases from ASST, checks priority, subject, applicant details and document readiness before LRA assignment. LRS can query citizen for more details, and the citizen reply will appear in the LRS case view.

Query to Citizen / Query Inbox

For any step, LRS can ask the citizen for clarification or documents. Citizen response will be visible in the LRS case view, with WhatsApp/SMS notification for query and response.

Assign Case to LRA

LRS selects LRA based on availability/workload and category-process flow. WhatsApp/SMS notification is sent to the citizen and internal notification is sent to the assigned LRA.

Revert to ASST

If LRS finds gaps in ASST scrutiny, wrong category-process routing, missing documents or incomplete registration details, LRS can revert the case to ASST with remarks. ASST receives the case in returned queue and WhatsApp/SMS notification is sent for the action.

AI Chatbot + Case Summary

LRA gets AI chatbot support in dashboard and verification report view. AI summary and key points will show applicant details, land details, LRS instructions, pending deadline and checklist for verification. WhatsApp/SMS notification is triggered for query, report submission or return actions.

LRA Dashboard

LRA dashboard shows assigned cases, verification pending cases, deadlines, applicant details, location, documents, LRS/CO instructions, citizen query responses, reports submitted, cases reverted to LRS and cases returned by CO for re-verification.

LRA Verification

LRA verifies the case details, land/documents and factual points as required. If field visit is required, LRA will visit the location and record the observations.

Query to Citizen / Query Inbox

For any step, LRA can query the citizen for verification/document clarification. Citizen response will be visible in the LRA case view, with WhatsApp/SMS notification for query and response.

LRA Report

LRA uploads supporting photographs/documents wherever required, enters detailed remarks, selects recommendation such as approval/rejection/further inquiry and submits the verification report to CO. LRA can also query the citizen for more details, and WhatsApp/SMS notification is sent for every action.

Revert to LRS

If assignment details, LRS instructions, documents, location mapping or category details are incomplete, LRA can revert the case to LRS with remarks. LRS receives it in returned queue and WhatsApp/SMS notification is sent for the action.

AI Chatbot + Case Summary

CO gets AI chatbot support in dashboard and case view. AI summary and key points will show full case history, ASST scrutiny, LRS remarks, LRA findings, SLA breach and decision support points. WhatsApp/SMS notification is triggered for every decision, query, transfer or closure action.

CO Dashboard

CO dashboard will have everything: all case details, user-wise reports, ASST/LRS/LRA-wise reports, SLA monitoring, pending final decisions and overall workflow monitoring.

Review Applications

CO reviews the application details, documents, ASST scrutiny, LRS remarks, LRA verification report, supporting photographs/documents, citizen query responses and AI generated summary/key points. CO can query the citizen for more details from the case view.

Query to Citizen / Query Inbox

For any step, CO can ask citizen for more details before final order. Citizen response will be visible in the CO case view, with WhatsApp/SMS notification for query and response.

Revert to ASST / LRA

CO can revert cases based on source path. If the case came directly from ASST, CO can revert it to ASST. If the case came through LRA verification, CO can revert it to LRA for re-verification or clarification. WhatsApp/SMS notification is sent for every revert action.

Transfer / Reassignment

CO can forcefully transfer or reassign delayed, disputed or workload-sensitive cases to a different LRA. The reassigned LRA receives the case in the LRA dashboard and WhatsApp/SMS notification is sent for the action.

Case Closure

CO reviews the applications and closes the Application/Case. Final decision may include approval, rejection, return for further inquiry or other official order as per case purpose. WhatsApp/SMS notification is sent after closure.

Final Notification

System generates final order PDF, updates case status, sends WhatsApp/SMS acknowledgement to citizen and makes final order PDF and application receipt available for download.

Core citizen flow

Transparent tracking

Citizen can apply, track every stage, respond to officer queries and download the signed final order from one portal.

Office workflow

Role-based processing

ASST handles registration/scrutiny, LRS supervises and assigns, LRA verifies case facts and visits the field only if required, and CO handles transfer/reassignment and case closure.

Revert movement

Clear return paths

CO can revert to ASST or LRA based on the source path, LRA can revert to LRS, and LRS can revert to ASST with remarks.

AI layer

AI for every user

AI chatbot, AI summary and AI key points are available in every user dashboard and case view.
